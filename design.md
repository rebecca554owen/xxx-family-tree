# 家族图谱应用设计文档（design）

## 1. 视觉风格
- 方向：卷轴质感、古典高雅、低噪点信息表达。
- 主色：宣纸底 + 墨色文本 + 朱砂强调。
- 动效：克制，强调状态反馈清晰性。

## 2. 交互设计（当前实现）

### 2.1 单元卡交互
- 点击卡片空白：选中单元。
- 点击头像/姓名：选中个人（右侧详情联动）。
- 双人卡支持“互换”按钮，交换左右槽位。

### 2.2 关系线交互
- 关系线可点击高亮、可删除。
- 支持拖动端点进行重连。
- 关系线命中宽度增大，降低选线难度。

### 2.3 连线创建机制（本次重构）
- 每边 3 锚点，连接目标更易命中。
- 拖线方向决定关系类型：
- 左右侧连接：`sibling`
- 上下侧连接：`parent_child`
- 连接失败时展示明确提示，避免用户无反馈。
- 锚点有连接态与可连接态高亮反馈。

### 2.4 布局交互
- 全景模式：允许拖拽并持久化。
- 同代约束：Y 轴锁定一致。
- 防重叠：同代单元卡自动避让。

## 3. 视图规则
- 焦点树：2 跳邻域。
- 父系/母系：基于 `parent_child` 上游分支识别。
- 全景：全量节点与手工布局。

## 4. 头像体验策略
- 上传后即时预览。
- 上传失败降级保存，保证刷新后仍可见。
- 若 storage policy 正确，则对象存储正常落库。

## 5. 线条可读性策略（本次）
- 优先使用正交平滑线（smooth-step）提升结构感。
- 选中关系线有明确发光高亮。
- 线条层级高于节点背景，减少“线被盖住”的感知问题。
- 多连接场景下使用稳定锚点分配，减少既有关系线被打乱。
